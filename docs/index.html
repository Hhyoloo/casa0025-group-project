Project Summary

From heat mapping to cooling-priority screening

Problem Statement

Extreme heat is an increasing planning concern for London, but heat exposure is spatially uneven. Dense built-up areas with limited vegetation and poorer access to cool spaces may experience higher cooling needs than greener neighbourhoods under the same city-wide heat warning. The application answers one practical question: where should cooling interventions be prioritised?

End User

The tool is designed for London borough officers, climate adaptation teams, urban greening practitioners and community-facing planners. It does not replace detailed thermal-comfort modelling; instead, it provides a transparent neighbourhood-scale screening tool for comparing heat burden, cooling capacity, population exposure and cool-space access.

Planning Context

Policy background

The project is framed by The London Plan 2021, especially policies on managing heat risk, urban greening and fair access to climate adaptation resources.

Analytical Logic

Cool Refuge analytical framework

The project uses a multi-criteria spatial index framework rather than a predictive machine-learning model. Each layer is converted to a comparable 0–1 scale. Higher LST, stronger built-surface intensity, higher population density and poorer cool-space access indicate greater cooling priority; higher NDVI indicates stronger cooling capacity and is therefore reversed in the risk-oriented score.

Interface

How users read the application

The interface is organised around layer comparison. Users can switch between LST, NDVI, built surface, population density, cool-space accessibility and composite CRI change. The legend updates with the active layer, while the side panel explains how colours should be interpreted and what each layer contributes to the planning question.

Code and Workflow

How it works

The repository should show the full workflow: Google Earth Engine preprocessing, raster harmonisation, composite-index construction, LSOA aggregation and Leaflet web mapping. 

// Google Earth Engine preprocessing example
var image = ee.ImageCollection('COPERNICUS/S2_SR_HARMONIZED')
  .filterBounds(london)
  .filterDate('2025-06-01', '2025-08-31')
  .filter(ee.Filter.lt('CLOUDY_PIXEL_PERCENTAGE', 20))
  .median();

var ndvi = image.normalizedDifference(['B8', 'B4'])
  .rename('NDVI')
  .clip(london);
// Normalisation and composite index logic
function normalise(img, region) {
  var minMax = img.reduceRegion({
    reducer: ee.Reducer.minMax(),
    geometry: region,
    scale: 30,
    bestEffort: true
  });
  return img.unitScale(minMax.get('min'), minMax.get('max'));
}

// Higher NDVI means more cooling capacity, so it is reversed in the risk score.
var cri = lstNorm.multiply(0.4)
  .add(ndbiNorm.multiply(0.2))
  .add(ndviNorm.multiply(-0.2))
  .add(popNorm.multiply(0.2));
// Leaflet layer control example
const map = L.map('map').setView([51.5072, -0.1276], 10);
L.tileLayer('https://{s}.basemaps.cartocdn.com/light_all/{z}/{x}/{y}{r}.png', {
  attribution: '&copy; OpenStreetMap contributors &copy; CARTO'
}).addTo(map);

L.geoJSON(criChange, {
  style: feature => ({
    fillColor: getCriColour(feature.properties.cri_change),
    weight: 0.4,
    fillOpacity: 0.72
  })
}).addTo(map);

Interpretation

Planning value

The application is most useful as an exploratory planning screen. Red areas in the composite layer indicate neighbourhoods where relative cooling conditions have worsened or where high heat burden is combined with lower cooling capacity and higher exposure. Blue areas indicate relative improvement or lower priority. These outputs can inform where to investigate tree planting, shade provision, cool-route planning, public-space upgrades and targeted community communication during heat events.

Critical Reflection

Limitations and future development 

<article class="info-card"><h3>Temporal consistency</h3><p>Not every layer is available for the same year. Population and accessibility layers may need to be interpreted as contextual rather than directly time-matched indicators.</p></article>
<article class="info-card"><h3>Index assumption</h3><p>The CRI is a relative composite index. It does not directly measure human thermal comfort, which would also require humidity, wind, shade, activity level and exposure duration.</p></article>
<article class="info-card"><h3>Spatial aggregation</h3><p>LSOA-level summaries are useful for planning comparison, but they can smooth local microclimate variation within neighbourhoods.</p></article>
